Desarrollar y Optimizar

Previniendo ineficiencias desde la primera línea de código con la filosofía "shift-left".

watsonx Code Assistant for Z

Optimización de código COBOL impulsada por IA directamente en el IDE del desarrollador.

⚙️ Características Clave

  • ✓Análisis en Tiempo de Ejecución: Identifica ineficiencias que solo se manifiestan con datos reales.
  • ✓Recomendaciones Priorizadas: Clasifica problemas por impacto para enfocar los esfuerzos.
  • ✓Integración con VS Code: Se integra fluidamente en el flujo de trabajo del desarrollador moderno.

🎯 Caso de Uso Principal

Un desarrollador está modernizando un programa batch de COBOL que consume muchos recursos. Usando la extensión en VS Code, la IA analiza el código y recomienda reescribir un bucle de procesamiento de datos, reduciendo el consumo de CPU en un 15% antes de que el código llegue a producción.

Diagnosticar y Analizar

Encontrando la causa raíz de problemas complejos con precisión quirúrgica.

Application Performance Analyzer (APA)

Diagnóstico para aplicaciones en ejecución con impacto mínimo.

⚙️ Características Clave

  • ✓Mapeo del Programa Fuente: Correlaciona el rendimiento directamente con las líneas de código exactas.
  • ✓Análisis No Intrusivo: Utiliza muestreo estadístico para minimizar el overhead en producción.
  • ✓Soporte Amplio: Compatible con CICS, Db2, IMS, Batch y múltiples lenguajes.

🎯 Caso de Uso Principal

Los usuarios reportan lentitud en una transacción CICS. Un analista usa APA y descubre que el 80% del tiempo de respuesta es una espera en una llamada a Db2. El informe señala la sentencia `EXEC SQL` exacta, permitiendo una solución rápida y dirigida.

CICS Performance Analyzer (PA) & Interdependency Analyzer (IA)

Análisis histórico profundo y descubrimiento de dependencias en CICS.

⚙️ Características Clave

  • ✓Análisis Histórico (PA): Compara el rendimiento antes y después de cambios para validar mejoras.
  • ✓Análisis de Afinidad (IA): Identifica dependencias que impiden el enrutamiento dinámico.
  • ✓Visualización de Dependencias (IA): Facilita la comprensión de arquitecturas complejas.

🎯 Caso de Uso Principal

Un equipo planea habilitar el enrutamiento dinámico para una aplicación. CICS IA identifica una afinidad oculta con un área de almacenamiento temporal. Una vez resuelta, CICS PA se usa para confirmar que el rendimiento no se ha degradado tras el cambio.

Monitorizar y Alertar

El sistema nervioso del mainframe, proporcionando una visión 24/7 de la salud del sistema.

IBM OMEGAMON Suite

Monitorización continua y en tiempo real de z/OS y sus subsistemas críticos.

⚙️ Características Clave

  • ✓Visión Unificada: Paneles de control para z/OS, CICS, Db2, IMS, Storage y Redes.
  • ✓Detección de Anomalías con IA: Aprende el comportamiento normal y alerta sobre desviaciones sutiles.
  • ✓Alertas Proactivas ("Situaciones"): Permite definir condiciones que desencadenan alertas y acciones automatizadas.

🎯 Caso de Uso Principal

OMEGAMON AI for CICS detecta una degradación en el tiempo de respuesta de una transacción. La alerta se activa antes de que se viole un SLA, permitiendo al equipo de operaciones investigar y resolver el problema (quizás usando APA) antes de que los usuarios finales se vean afectados.

Estrategizar y Prevenir

Pasando de la analítica histórica a la prevención de incidentes con AIOps.

Z Performance & Capacity Analytics (IZPCA)

Visión a largo plazo para la planificación de capacidad y gestión de costos.

⚙️ Características Clave

  • ✓Análisis de Tendencias Históricas: Utiliza datos SMF para identificar patrones de crecimiento.
  • ✓Previsión de Capacidad: Predice las necesidades futuras de recursos para evitar compras reactivas.
  • ✓Gestión de Costos: Ayuda a optimizar los gastos de software.

🎯 Caso de Uso Principal

Un planificador de capacidad utiliza IZPCA para analizar el crecimiento del consumo de MSU durante los últimos 2 años. El informe de previsión indica que se necesitará una actualización de CPU en 9 meses, dando tiempo suficiente para planificar y presupuestar la compra.

Z Anomaly Analytics (IZAA) & IntelliMagic Vision

Prevención proactiva de incidentes y analítica de rendimiento holística.

⚙️ Características Clave

  • ✓Detección de Anomalías (IZAA): Aprende el comportamiento normal y detecta desviaciones sutiles en tiempo real.
  • ✓Health Insights (IntelliMagic): Califica la salud de la infraestructura e identifica riesgos ocultos.
  • ✓Resolución Acelerada: Ambas herramientas guían a los analistas hacia la causa raíz.

🎯 Caso de Uso Principal

IZAA emite una alerta sobre un comportamiento anómalo en los logs de Db2. El equipo utiliza IntelliMagic Vision para investigar y descubre una calificación de salud deficiente en los buffer pools, identificando un riesgo de rendimiento antes de que cause una interrupción del servicio.

Flujo de Trabajo: De la Alerta a la Resolución

Vea cómo las herramientas de IBM colaboran en un escenario real, demostrando el poder de un ecosistema integrado.

1. Detección Proactiva

OMEGAMON AI for CICS detecta una degradación del 30% en el tiempo de respuesta de transacciones críticas, emitiendo una alerta basada en una desviación del comportamiento normal.

📈
🔬

2. Aislamiento de Causa Raíz

Un analista usa APA. El informe revela que el 70% del tiempo es espera de Db2 y el mapeo de código fuente señala las sentencias `EXEC SQL` exactas en un programa COBOL.

3. Optimización Asistida por IA

El desarrollador utiliza watsonx Code Assistant for Z en VS Code, que recomienda reescribir la consulta SQL para usar un índice de manera más efectiva.

💻
📊

4. Validación de la Mejora

CICS PA compara informes de antes y después, confirmando una reducción drástica en la espera de Db2. APA verifica que las sentencias SQL ya no son un cuello de botella.

5. Revisión Estratégica

Meses después, IntelliMagic Vision e IZPCA identifican una tendencia sistémica de acceso ineficiente a la base de datos, impulsando una iniciativa de formación para prevenir problemas futuros.
